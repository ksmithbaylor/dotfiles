Autenticazione token

Pulse supporta il sistema di autenticazione token RSA SecurID. L'autenticazione token, denominata anche autenticazione a due fattori, è un metodo per la generazione di credenziali di autenticazione molto difficili da compromettere. Un codice token è un numero che viene generato simultaneamente sul client e sul server. L'utente crea un PIN personale e quindi il nome utente e il PIN più il codice token corrente gli consentono di accedere al server. Un PIN più un codice token formano un passcode. Un codice token scade dopo un breve intervallo di tempo, in genere di un minuto, e viene quindi sostituito da uno nuovo.

Al primo utilizzo del sistema, Pulse richiede all'utente di creare un PIN. Se l'amministratore ha già fornito un PIN, viene visualizzata una richiesta per cambiarlo. Se l'amministratore di rete richiede il cambiamento periodico del PIN, Pulse richiede all'utente un nuovo PIN. In genere, è necessario fornire il codice token corrente quando si cambia il PIN. In alcuni casi, dopo l'inserimento del passcode, viene richiesto l'inserimento del codice token successivo prima di poter accedere alla rete. Attendere che il dispositivo token passi a visualizzare un nuovo codice token, quindi immetterne il numero per procedere.

Autenticatori

Il dispositivo o il software che genera il token è denominato autenticatore. Pulse supporta la gamma completa di autenticatori hardware SecurID.

Gli autenticatori hardware (fisici) RSA SecurID visualizzano un nuovo codice token ogni 60 secondi. Quando si esegue l'accesso, è necessario disporre di un dispositivo di autenticazione a chiave o a scheda che generi il token. Nella finestra di accesso di Pulse, occorre indicare il nome utente e il PIN più il codice token corrente. È possibile archiviare le credenziali nome utente/password di MacOS e i certificati digitali in un autenticatore USB. Quando il dispositivo viene connesso al computer, si attiva l'immissione automatica dei codici token e Pulse può accedere automaticamente a tali codici direttamente dal dispositivo dopo l'immissione del PIN da parte dell'utente.
