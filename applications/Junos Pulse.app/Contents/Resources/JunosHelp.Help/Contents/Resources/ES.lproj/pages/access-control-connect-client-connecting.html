Conectar a una red

Debe tener al menos una conexión en la lista del Panel de conexiones para poder conectarse a una red. Los indicadores de la conexión que va a ver dependerán de su entorno de acceso a la red.

Nota: Para utilizar Junos Pulse con una red inalámbrica, debe configurar primero los ajustes de la red inalámbrica de su equipo a través de Preferencias del sistema.

Para usar una conexión definida para conectarse a una red:

  1. En la lista de conexiones, haga clic en Conectar en la conexión que desee establecer.
  2. Responda las solicitudes de información como nombre de usuario y contraseña, territorio, rol, etc. Las solicitudes que va a ver dependerán de la forma en la que el administrador de red haya configurado su entorno de acceso.
  3. Aparecerá un cuadro de diálogo con el estado de conexión. Si el intento de conexión se ha realizado correctamente, podrá hacer clic en Cerrar o esperar un momento hasta que el cuadro de diálogo se cierre automáticamente.

    Si hay un problema con el intento de conexión, un cuadro de diálogo de corrección va a mostrar información sobre lo que puede hacer para resolver el problema. También puede aparecer un cuadro de diálogo de corrección, que muestra el progreso de las medidas que ha tomado Junos Pulse de forma automática para solucionar el problema.

Tras hacer clic en el botón Conectar, puede que necesite responder a las siguientes solicitudes:

  • Certificado: si Pulse necesita comunicarse con un servidor de certificados y el administrador de red ha configurado más de uno, se le pedirá que elija un servidor. Un certificado emitido por una autoridad de certificación verifica que el recurso de red al que se conecta es válido. Si el certificado procede de una fuente fiable, se acepta automáticamente y no verá la solicitud de certificado. Si hay un problema con el certificado, es posible que se le pregunte si desea aceptar el certificado y realizar la conexión.
  • Territorio: un territorio define el recurso de red al que puede acceder. Si tiene acceso a más de un territorio, debe seleccionar uno.
  • Credenciales: su nombre de usuario y contraseña o nombre de usuario y código token determinan su identidad para acceder al dispositivo. Puede que también se le pida un nombre de usuario y contraseña secundarios, así como un nombre de usuario y contraseña para un servidor proxy. Su entorno de autenticación puede solicitarle periódicamente que cambie su contraseña y su número PIN de token.
  • Rol: un rol define los parámetros de la sesión (valores de tiempo de espera, el número de sesiones simultáneas permitido, etc.), los ajustes de personalización y las características de acceso activadas. El administrador de red puede permitirle escoger el rol que desee al conectarse.
  • Sesiones simultáneas: si inicia sesión en el sistema más de una vez (por ejemplo, utiliza las mismas credenciales para iniciar sesión desde dos equipos diferentes), es posible que el sistema le solicite cerrar la sesión actual antes de comenzar una nueva.
  • Acciones de corrección: puede que se le pida llevar a cabo acciones para que el equipo cumpla las directivas de seguridad.

En cada pantalla de inicio de sesión, puede aparecer la casilla de verificación Guardar ajustes. (El administrador puede desactivar esta función). Si activa la casilla de verificación, la próxima vez que inicie sesión no se le pedirá esa información. Si guarda los ajustes, puede usar la función Olvidar ajustes guardados para que se vuelva a solicitar la información de inicio de sesión. La función Guardar ajustes le permite guardar la siguiente información:

  • Aceptación del certificado
  • Selección del certificado
  • Territorio
  • Nombre de usuario y contraseña
  • Nombre de usuario/contraseña de proxy
  • Nombre de usuario/contraseña secundarios
  • Rol

Los pasos que se realizan después de responder a todas las solicitudes de conexión dependen de las directivas de acceso que el administrador de red haya configurado y del tipo de dispositivo de acceso a red. El proceso de conexión puede incluir las siguientes tareas:

  • Actualizaciones de software: Junos Pulse se puede actualizar automáticamente al establecer la conexión. El sistema podría recibir software Junos Pulse actualizado o podría recibir conexiones adicionales.
  • Host Checker: Host Checker examina su punto final para determinar si cumple las directivas de seguridad de su organización. Por ejemplo, Host Checker puede verificar si tiene un software antivirus y firmas de virus actualizadas, el estado del cortafuegos personal y la presencia de malware y spyware. Estos ajustes los define el administrador de red. Si Host Checker encuentra un problema, puede mostrar un mensaje que describe los pasos que debe seguir para corregir el problema o iniciar operaciones de corrección automáticamente.
