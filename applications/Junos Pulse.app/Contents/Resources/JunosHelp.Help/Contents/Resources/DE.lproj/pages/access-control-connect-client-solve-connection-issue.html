Junos Pulse-Verbindungsproblem beheben

Sie können die folgenden Fehlerbehebungsinformationen verwenden, um Verbindungsprobleme zu beheben.

Problem

Beschreibung und Lösungsvorschläge
Pulse hat Wireless-Verbindungen deaktiviert und zeigt Deaktiviert
- Wired-Verbindung aktiv an.		 Der Pulse-Administrator kann Pulse-Verbindungen konfigurieren, sodass die Wireless-Verbindungen nur in Kraft treten, wenn keine Wired-Verbindung zur Verfügung steht. Sobald Sie Ihren Computer mit einer Wired-Verbindung verbinden, beispielsweise durch Anschluss an eine Docking-Station, unterbricht Pulse die Wireless-Verbindung und verwendet stattdessen die Wired-Verbindung.

Nach dem Installieren von Pulse und dem Deinstallieren von Odyssey Access Client (OAC) kann Ihr System keine Verbindung mehr zu einem Wireless-Netzwerk herstellen.

Junos Pulse ersetzt die von OAC bereitgestellten Zugangsdienste. OAC verwaltet die Wireless-Verbindungen eines Computers, und Pulse ist auf die von MacOS bereitgestellten Verbindungsfunktionen angewiesen. Nach dem Deinstallieren von OAC müssen Sie die Wireless-Netzwerkeigenschaften Ihres Computers festlegen, um Wireless-Netzwerke zu verwalten (Systemeinstellungen > Netzwerk).

Während der Anmeldung erscheint folgende Meldung:

Sie sind im Begriff, einen nicht vertrauenswürdigen Server zu authentifizieren. Akzeptieren Sie das Zertifikat für diese Verbindung?

                          

Junos Pulse kann die Identität des Servers nicht verifizieren. Dieser Fehler sowie sämtliche Zertifikatfehler bedeuten, dass Pulse nicht sicherstellen kann, ob Sie eine Verbindung zu einem vertrauenswürdigen Server herstellen. Möglicherweise wurde das Serverzertifikat widerrufen oder ist abgelaufen. Es wurde möglicherweise von einer Zertifizierungsstelle ausgestellt, die von Pulse nicht erkannt wird, z. B. wenn Ihre Organisation ein selbst signiertes Zertifikat verwendet. Falls Ihr Netzwerkadministrator diese Genehmigung erteilt hat, können Sie fortfahren und eine Verbindung zum Server herstellen. Sie sollten diesen Schritt jedoch nur ausführen, wenn der Netzwerkadministrator Sie tatsächlich angewiesen hat, den Zertifikatfehler zu ignorieren.

Nach der Anmeldung erscheint folgende Meldung:

Der Netzwerkzugriff ist aufgrund eines Kompatibilitätsproblems eingeschränkt.

Ihr Endgerät entspricht mindestens einer Anforderung für Ihre Netzwerkzugriffsrichtlinien nicht. Falls die Informationen zur Reparatur von Pulse nicht genügend Informationen zur Lösung des Problems enthalten, wenden Sie sich an Ihren Netzwerkadministrator. Wenn Sie das Problem beheben, stellt Junos Pulse automatisch eine Verbindung zum Netzwerk her.

Während der Anmeldung erscheint folgende Meldung:

Die Anmeldeinformationen sind ungültig. Versuchen Sie es noch einmal.

Falls Sie das Kontrollkästchen Einstellungen speichern beim Herstellen einer Pulse-Verbindung aktiviert haben, werden die bereitgestellten Anmeldeinformationen bei jeder Herstellung dieser Verbindung ohne Aufforderung zur Eingabe der Anmeldeinformationen verwendet. Junos Pulse erkennt jedoch nicht, wenn Sie das Netzwerkkennwort geändert haben, die gespeicherten Einstellungen werden weiter verwendet. Dies kann zum Fehler „Die Anmeldeinformationen sind ungültig“ führen. Klicken Sie zur Behebung dieses Problems auf die Schaltfläche Datei > Verbindungen > Gespeicherte Einstellungen vergessen.

Bei der nächsten Anmeldung werden Sie zur Eingabe der Anmeldeinformationen aufgefordert. Sie können zudem Ihre neuen Anmeldeinformationen festlegen. Falls Sie das Kontrollkästchen Einstellungen speichern aktivieren, werden die neuen Anmeldeinformationen gespeichert.

Das Symbol des Pulse-Menüs zeigt den Status „Fehlgeschlagen“ an:

Das Symbol der Pulse-Verbindungsliste zeigt den Status „Fehlgeschlagen“ an:

Wenn die Verbindung fehlgeschlagen ist, öffnen Sie Pulse und klicken auf den Pfeil neben dem Verbindungsnamen, um detaillierte Informationen zum Verbindungsstatus anzuzeigen. Der Abschnitt Details enthält den spezifischen Fehler, auf den Sie klicken können, um ein Fenster zu öffnen, das eine detaillierte Beschreibung des Fehlers enthält.

Ihr Anmeldeversuch schlägt fehl, wenn Sie den Zielbereich als Teil Ihres Benutzernamens angeben. Beispiel:

benutzername@realm

Die Konvention benutzername@realm wird jedoch in Junos Pulse nicht unterstützt. Falls Sie Zugang zu mehr als einem Bereich haben, fordert Juno Pulse Sie auf, den gewünschten auszuwählen.

Verwandte Themen

  • Junos Pulse-Verbindungsstatus
  • Gespeicherte Einstellungen vergessen
  • Verbindungseigenschaften anzeigen
