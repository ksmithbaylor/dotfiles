Verbindung mit einem Netzwerk herstellen

Im Verbindungsbereich muss mindestens eine Verbindung aufgeführt sein, bevor Sie eine Verbindung zu einem Netzwerk herstellen können. Die angezeigten Verbindungsaufforderungen hängen von der Netzwerkzugriffsumgebung ab.

Hinweis: Um Junos Pulse mit einem Wireless-Netzwerk verwenden zu können, müssen Sie möglicherweise zunächst das Wireless-Netzwerk des Computers in den Systemeinstellungen konfigurieren.

So verwenden Sie eine definierte Verbindung zum Herstellen einer Netzwerkverbindung:

  1. Klicken Sie in der Verbindungsliste für die Verbindung, die Sie herstellen möchten, auf Verbinden.
  2. Geben Sie bei Aufforderung die Informationen wie etwa Benutzername und Kennwort, Bereich, Rolle usw. ein. Die angezeigten Aufforderungen hängen von der Konfiguration der Zugriffsumgebung durch den Netzwerkadministrator ab.
  3. Ein Dialogfeld für den Verbindungsstatus wird angezeigt. Bei erfolgreichem Verbindungsversuch können Sie auf Schließen klicken oder einen Augenblick warten, bis das Dialogfeld automatisch geschlossen wird.

    Falls beim Verbindungsversuch ein Problem aufgetreten ist, werden in einem Reparaturdialogfeld Informationen zu möglichen Maßnahmen zur Behebung oder ein separates Reparaturdialogfeld angezeigt, in dem der Fortschritt der Maßnahmen angezeigt wird, die Junos Pulse automatisch zur Behebung des Problems ergriffen hat.

Nach dem Klicken auf die Schaltfläche Verbinden müssen Sie möglicherweise auf folgende Eingabeaufforderungen reagieren:

  • Zertifikat – Falls Pulse mit einem Zertifikatserver kommunizieren muss und der Netzwerkadministrator mehr als einen Server konfiguriert hat, werden Sie zur Auswahl eines Servers aufgefordert. Ein von einer Zertifizierungsstelle ausgestelltes Zertifikat prüft, ob die Netzwerkressource gültig ist, zu der die Verbindung hergestellt werden soll. Falls das Zertifikat von einer vertrauenswürdigen Quelle stammt, wird es automatisch akzeptiert, und die Zertifikataufforderung wird nicht angezeigt. Bei Problemen mit dem Zertifikat werden Sie möglicherweise gefragt, ob Sie das Zertifikat akzeptieren und die Verbindung weiter verwenden möchten.
  • Bereich – Ein Bereich definiert die Netzwerkressourcen, auf die Sie zugreifen können. Falls Sie auf mehr als einen Bereich zugreifen, müssen Sie einen auswählen.
  • Anmeldeinformationen – Anhand Ihres Benutzernamens und Kennwortes oder des Benutzernamens und Token-Codes übermitteln Sie Ihre Identität an das Zugriffsgerät. Sie werden möglicherweise auch aufgefordert, einen sekundären Benutzernamen und ein Kennwort sowie einen Benutzernamen und ein Kennwort für einen Proxyserver einzugeben. Sie werden möglicherweise regelmäßig von der Authentifizierungsumgebung aufgefordert, das Kennwort und die Token-PIN zu ändern.
  • Rolle – Eine Rolle definiert Sitzungsparameter (Zeitüberschreitungswerte, Anzahl der zulässigen gleichzeitigen Sitzungen usw.), individuelle Einstellungen und aktivierte Zugriffsfunktionen. Ihr Netzwerkadministrator kann das Auswählen der gewünschten Rolle beim Verbinden zulassen.
  • Gleichzeitige Sitzungen – Falls Sie sich mehrmals am System anmelden, wenn Sie z. B. dieselben Anmeldeinformationen zur Anmeldung an zwei verschiedenen Computern verwenden, werden Sie möglicherweise aufgefordert, die aktuelle Sitzung zu beenden, bevor Sie eine neue beginnen.
  • Reparaturaktionen – Sie werden möglicherweise aufgefordert, Aktionen auszuführen, damit Ihr Endgerät die Sicherheitsrichtlinien erfüllt.

Das Kontrollkästchen „Einstellungen speichern“ wird möglicherweise auf jedem Anmeldebildschirm angezeigt. (Der Administrator kann diese Funktion deaktivieren.) Falls Sie dieses Kontrollkästchen aktivieren, werden Sie bei der nächsten Anmeldung nicht nach diesen Informationen gefragt. Falls Sie die Einstellungen speichern, können Sie die Funktion „Gespeicherte Einstellungen vergessen“ verwenden, damit Sie wieder nach Anmeldeinformationen gefragt werden. Die Funktion „Einstellungen speichern“ ermöglicht das Speichern folgender Informationen:

  • Annahme des Zertifikats
  • Zertifikatauswahl
  • Bereich
  • Benutzername und Kennwort
  • Proxybenutzername/Kennwort
  • Sekundärer Benutzername/Kennwort
  • Rolle

Die Schritte nach der Beantwortung aller Verbindungsaufforderungen sind von den Zugriffsrichtlinien, die der Netzwerkadministrator konfiguriert hat, und vom Typ des Netzwerkzugriffsgeräts abhängig. Der Verbindungsprozess kann die folgenden Aufgaben umfassen:

  • Softwareaktualisierungen – Junos Pulse kann bei bestehender Verbindung automatisch aktualisiert werden. Das System erhält aktualisierte Junos Pulse-Software, oder Sie erhalten zusätzliche Verbindungen.
  • Host Checker — Host Checker testet Ihr Endgerät, um zu ermitteln, ob es den Sicherheitsrichtlinien des Unternehmens entspricht. Host Checker sucht z. B. nach Antivirussoftware und aktuellen Virensignaturen, dem persönlichen Firewallstatus und nach vorhandener Malware und Spyware. Diese Einstellungen werden vom Netzwerkadministrator definiert. Falls Host Checker ein Problem findet, wird entweder eine Nachricht angezeigt, in der die Schritte beschrieben werden, die zur Behebung des Problems ausgeführt werden müssen, oder die Korrekturvorgänge werden automatisch gestartet.

Verwandte Themen

  • Junos Pulse-Benutzeroberfläche
  • Junos Pulse-Verbindungsproblem beheben
  • Gespeicherte Einstellungen vergessen
