Se connecter à un réseau

Au moins une connexion doit être affichée dans le volet Connexions pour que vous puissiez vous connecter à un réseau. Les invites de connexion affichées dépendent de votre environnement d’accès réseau.

Remarque : pour utiliser Junos Pulse avec un réseau sans fil, il peut être nécessaire de configurer au préalable les paramètres réseau sans fil de votre ordinateur dans les Préférences système.

Pour utiliser une connexion définie en vue de se connecter à un réseau :

  1. Dans la liste Connexions, cliquez sur Connexion pour la connexion à établir.
  2. Répondez aux demandes d’informations telles que nom d’utilisateur et mot de passe, domaine, rôle, etc. Les invites affichées dépendent de la manière dont l’administrateur réseau a configuré votre environnement d’accès.
  3. Une boîte de dialogue d’état de la connexion s’affiche. Si la tentative de connexion a réussi, vous pouvez cliquer sur Fermer ou attendre la fermeture automatique de la boîte de dialogue.

    En cas de problème de connexion, une boîte de dialogue de correction affiche ce que vous pouvez faire pour résoudre le problème ou une boîte de dialogue de correction distincte affiche la progression des mesures prises automatiquement par Junos Pulse pour résoudre le problème.

Après avoir cliqué sur le bouton Connexion, vous devrez peut-être répondre aux invites décrites ci-dessous.

  • Certificat : si Pulse doit communiquer avec un serveur de certificats et que votre administrateur réseau a configuré plusieurs serveurs, il vous est demandé de choisir un serveur. Un certificat émis par une autorité de certification vérifie que la ressource réseau à laquelle vous vous connectez est valide. Si le certificat provient d’une source certifiée, il est accepté automatiquement et l’invite de certificat ne s’affiche pas. En cas de problème lié au certificat, vous serez peut-être invité à confirmer que vous acceptez le certificat et souhaitez poursuivre la connexion.
  • Domaine : un domaine définit les ressources réseau auxquelles vous pouvez accéder. Si vous avez accès à plusieurs domaines, vous devez en sélectionner un.
  • Données d’identification : vos nom d’utilisateur et mot de passe ou vos nom d’utilisateur et code de jeton établissent votre identité au dispositif d’accès. Un nom d’utilisateur et un mot de passe secondaires ou un nom d’utilisateur et un mot de passe de serveur proxy peuvent également vous être demandés. Votre environnement d’authentification peut vous inviter périodiquement à modifier votre mot de passe ou votre code PIN de jeton.
  • Rôle : un rôle définit les paramètres de session (valeurs de délai d’attente, nombre de sessions simultanées autorisées, etc.), les paramètres de personnalisation et les fonctions d’accès activées. Votre administrateur réseau peut vous permettre de choisir le rôle souhaité lorsque vous vous connectez.
  • Sessions simultanées : si vous vous connectez au système plusieurs fois, par exemple, si vous utilisez les mêmes données d’identification pour vous connecter à partir de deux ordinateurs différents, il peut vous être demandé de mettre fin à la session en cours avant d’en commencer une autre.
  • Actions de correction : il peut vous être demandé d’effectuer des actions pour rendre votre point final conforme aux règles de sécurité.

La boîte de dialogue Enregistrer les paramètres peut s’afficher dans chaque écran de connexion. (Votre administrateur peut désactiver cette fonction.) Si vous cochez la case, ces informations ne vous seront plus demandées la prochaine fois que vous vous connecterez. Si vous enregistrez les paramètres, vous pouvez utiliser la fonction Effacer les paramètres enregistrés pour que les informations de connexion vous soient demandées aux prochaines connexions. La fonction Enregistrer les paramètres vous permet d’enregistrer les informations suivantes :

  • Validation du certificat
  • Sélection du certificat
  • Domaine
  • Nom d’utilisateur et mot de passe
  • Nom d’utilisateur/mot de passe du serveur proxy
  • Nom d’utilisateur/mot de passe secondaires
  • Rôle

Les étapes qui suivent vos réponses aux invites de connexion dépendent des stratégies d’accès configurées par l’administrateur réseau et du type de dispositif d’accès au réseau. Le processus de connexion peut inclure les tâches suivantes :

  • Mises à jour de logiciels : Junos Pulse peut être automatiquement mis à jour à la connexion. Votre système peut recevoir le logiciel Junos Pulse mis à jour ou vous pouvez recevoir des connexions supplémentaires.
  • Host Checker : Host Checker examine votre point final pour déterminer s’il est conforme aux règles de sécurité de votre organisation. Par exemple, Host Checker peut contrôler la présence de logiciels antivirus et de signatures de virus à jour, l’état du pare-feu personnel, ou encore la présence de logiciels espions et de programmes malveillants. Ces paramètres sont définis par l’administrateur réseau. Si Host Checker détecte un problème, il peut afficher un message décrivant les étapes à suivre pour résoudre le problème ou lancer des opérations de correction automatiquement.
