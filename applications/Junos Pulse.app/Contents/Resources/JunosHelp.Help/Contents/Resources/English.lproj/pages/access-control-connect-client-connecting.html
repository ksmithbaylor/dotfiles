Connect to a Network

You must have at least one connection listed in the Connections pane before you can connect to a network. The connection prompts you see depend on your network access environment.

Note: To use Junos Pulse with a wireless network, you might need to first configure your computer's wireless network settings through System Preferences.

To use a defined connection to connect to a network:

  1. In the Connections list, click Connect for the connection you want to establish.
  2. Respond to the prompts for information such as username and password, realm, role and so on. The prompts you see depend on how your network administrator has configured your access environment.
  3. A connection status dialog box appears. If the connection attempt was successful, you can click Close or wait a moment for the dialog box to close automatically.

    If there was a problem with the connection attempt, a remediation dialog box shows information on what you can do to resolve the issue or a separate remediation dialog box appears and shows the progress of measures that Junos Pulse has started automatically to fix the problem.

After you click the Connect button, you might need to respond to the following prompts:

  • Certificate — If Pulse needs to communicate with a certificate server, and your network administrator has configured more than one server, you are prompted to choose a server. A certificate issued by a certificate authority verifies that the network resource you are connecting to is valid. If the certificate is from a trusted source, it is automatically accepted and you do not see the certificate prompt. If there is a problem with the certificate, you might be asked if you want to accept the certificate and proceed with the connection.
  • Realm — A realm defines the network resources that you can access. If you have access to more than one realm, you need to select one.
  • Credentials — Your username and password or username and token code, establish your identity to the access device. You might also be prompted for a secondary username and password and a username and password to a proxy server. Your authentication environment might periodically prompt you to change your password or your token PIN number.
  • Role — A role defines session parameters (timeout values, the number of concurrent sessions allowed, and so on), personalization settings and enabled access features. Your network administrator can allow you to choose the role you want when you connect.
  • Concurrent sessions — If you log in to the system more than once, for example, you use the same credentials to log in from two different computers, you might be prompted to end the current session before beginning a new one.
  • Remediation actions — You might be prompted to perform actions to bring your endpoint into compliance with security policies.

A Save Settings check box might appear on each log in screen. (Your administrator can disable this feature.) If you enable the check box, you are not prompted for that information the next time you log in. If you save settings, you can use the Forget Saved Settings feature to return to being prompted for log in information. The Save Settings feature enables you to save the following information:

  • Certificate acceptance
  • Certificate selection
  • Realm
  • Username and password
  • Proxy username/password
  • Secondary username/password
  • Role

The steps that take place after you respond to all of the connection prompts depend on the access polices that your network administrator has configured and on the type of network access device. The connection process can include the following tasks:

  • Software updates — Junos Pulse can be automatically updated at connect time. Your system might receive updated Junos Pulse software or you might receive additional connections.
  • Host Checker — Host Checker examines your endpoint to determine if it meets your organization's security policies. For example, Host Checker might check for anti virus software and up-to-date virus signatures, for personal firewall status, and for the presence of malware and spyware. These settings are defined by your network administrator. If Host Checker finds a problem, it can display a message that describes the steps you must take to fix the problem or initiate remediation operations automatically.
