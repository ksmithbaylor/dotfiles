Token Authentication

Pulse supports the RSA SecurID token authentication system. Token authentication, also called two-factor authentication, is a method of generating authentication credentials that are very difficult to compromise. A token code is a number that is generated simultaneously on both the client and server. The user creates a personal identification number (PIN) and then the username and the PIN plus the current token code permit the user to login to the server. A PIN plus a token code forms a passcode. A token code expires after a short interval, typically a minute, and is then replaced by a newly generated token code.

The first time you use the system, Pulse prompts you to create a PIN. If your administrator has already provided a PIN, you are prompted to change it. If your network administrator requires you to change your PIN periodically, Pulse prompts you for a new PIN. Typically, you must provide the current token code when you change your PIN. In some cases, after entering your passcode, you are prompted to enter the next token code before you can access the network. Just wait until your token device changes to a new token code and then enter that number to proceed.

Authenticators

The device or software that generates the token is called an authenticator. Pulse supports the full range of SecurID hardware authenticators.

RSA SecurID hardware (physical) authenticators display a new token code every 60 seconds. You must have the card or key fob that generates the token with you when you login. At the Pulse login window, you provide your username and your PIN plus the current token code. A USB authenticator can store your MacOS username/password credentials and digital certificates. When the device is connected to your computer, it enables automatic token code entry and Pulse can automatically access token codes directly off the device after you provide your PIN.
