Troubleshoot a Junos Pulse Connection Issue

You can use the following troubleshooting information to help resolve connection issues.

Issue

Description and Resolution Suggestions
Pulse has disabled wireless connections and displays Disabled
- Wired Connection Active.		 The Pulse administrator can configure Pulse connections so that wireless connections work only when there is no wired connection available. As soon as you connect your computer to a wired connection, for example, by connecting it to a docking station, Pulse drops the wireless connection and uses the wired connection.

After installing Pulse and uninstalling Odyssey Access Client (OAC), your system can no longer connect to any wireless network.

Junos Pulse replaces the access services provided by OAC. However, OAC manages a computer's wireless connectivity and Pulse relies on the connectivity provided by MacOS. After uninstalling OAC, you must set your computer's wireless networking properties to manage wireless networks (System Preferences > Network).

During login, the following message appears:

You are about to authenticate to an untrusted server. Do you accept the certificate for this connection?

                          

Junos Pulse cannot verify the identity of the server. This error or any certificate error means that Pulse cannot ensure that you are connecting to a trusted server. The server certificate might have been revoked or it might have expired. It could have been issued by a certificate authority that is not recognized by Pulse, such as when your organization uses a self-signed certificate. If your network administrator has enabled this permission, you can choose to proceed and connect to the server but you should do so only if your network administrator has advised you to ignore the certificate error.

After login, the following message appears:

Limited network access due to compliance issue

Your endpoint does not meet one or more of the requirements that your network access policies require. If the Pulse Remediation information does not provide enough information to enable you to solve the issue, see your network administrator. After you fix the problem, Junos Pulse automatically connects you to the network.

During login, the following message appears:

Credentials were invalid. Please try again.

If you selected the Save Settings check box when you activated a Pulse connection, the credentials you provided are used every time you activate that connection without prompting you to enter login information. However, Pulse cannot detect when you change your network password, it continues to use the saved settings, and that can result in the "Credentials were invalid" error. To resolve this issue, click File > Connections > Forget Saved Settings.

The next time you connect, you are prompted for your login credentials, and you can specify your new login information. If you select the Save Settings check box, the new credential information will be saved.

The Pulse menu icon changes to the failed state:

The Pulse Connections list icon changes to the failed state:

When Pulse indicates that your network connection has failed, open Pulse and then click the arrow next to the connection name to display the detailed connection status. The Details section shows the specific error, which you can click to open a window that shows a detailed description of the error.

Your login attempt fails when you specify the target realm as part of your username. For example:

username@realm

The username@realm convention is not supported in Junos Pulse. If you have access to more than one realm, Pulse prompts you to select the one you want.
